Welcome to Qpca’s documentation!

This is the documentation of the Qpca package implemented using Qiskit SDK. The package is organized in 5 modules:

  • decomposition module: here there is QPCA Qpca class implementation. One of the main method is fit() to fit the Qpca model. In this context, fitting the model means building the circuit which encode an arbitrary input matrix that you provide as input and to perform phase estimation to encode the eigenvalues of the input matrix in the quantum registers. With eigenvectors_reconstruction() method you can reconstruct the original eigenvectors (and eigenvalues) by exploiting the StateVectorTomography class which implements state vector tomography algorithm. Finally, once you have reconstructed the eigenvectors and eigenvalues, you can also reconstruct the original input matrix using quantum_input_matrix_reconstruction(). If the output is not satisfying, you can think to increase the number of shots or the number of qubits for the phase estimation resolution.

  • quantumUtilities module: here there are the main quantum routines used by the Qpca algorithm. The main module contains the StateVectorTomography class that is used by Qpca to reconstruct the eigenvectors but can also be used as a stand alone class to reconstruct the statevector of an arbitrary quantum circuit (composed only of real amplitudes).

  • preprocessingUtilities module: in this module, you find the generate_matrix() method which is useful to generate an arbitrary random Hermitian input matrix for the Qpca algorithm.

  • postprocessingUtilities module: in this module, you find the general_postprocessing() method which is used by Qpca algorithm to recontruct the eigenvectors/eigenvalues using the information provided by quantum state tomography.

  • benchmark module: in this module, you find the Benchmark_Manager class that manages all the benchmarking by exploiting the implemented methods to benchmark the execution of Qpca algorithm.
